# OPC 高手创造赛评分维度对齐

## 赛题要求

OPC 高手创造赛要求围绕真实场景，提交一个完整、可展示、可复用的 OPC 作品，展现“一人 + 小浣熊”从需求到交付的全栈能力。

## 对齐点

### 1. 真实行业/工作场景

开源项目收藏、技术选型和知识复用是开发者、AI 工具用户、学生和小团队都真实存在的场景。GitHub Star 数量增长后，用户常常无法找回当时的收藏动机，也难以将收藏转化为项目、课程或工作中的可复用资产。

### 2. 完整交付

Star-DART OPC 不止生成一段摘要，而是交付完整办公链路：

- 定时轮询发现新增 Star。
- 读取 README、DeepWiki、Release、Issue 等资料。
- 生成项目档案。
- 更新资产台账。
- 构建数据看板。
- 生成周报和 PPT。
- 沉淀到知识库。

### 3. 体现办公小浣熊核心能力

本作品组合使用：

- 本地 Agent
- Skill
- 专家团
- research-synthesis
- docx/pdf
- xlsx
- data-dashboard
- pptx
- writing-workflow
- 云上知识库
- 本地文件索引
- 飞书 / Obsidian 第三方数据
- 定时任务

这不是简单单点使用，而是完整能力组合。

### 4. 可展示

可展示材料包括：

- dry-run 命令输出。
- 项目档案样例。
- 资产台账样例。
- 看板数据样例。
- 周报样例。
- PPT 大纲。
- 办公小浣熊实际操作截图。

### 5. 可复用

Star-DART OPC 可扩展到：

- 论文库
- AI 工具库
- 竞品情报库
- 课程资料库
- 团队技术雷达

## 建议强调的评审话术

不要说：

> 我做了一个 GitHub Star 自动整理脚本。

应该说：

> 我用办公小浣熊搭建了一套持续运行的开源项目情报工作流。它把 GitHub Star 这种碎片化收藏，转化为可检索、可汇报、可复用的知识资产，并用文档、表格、看板、PPT 和定时周报完成完整交付。
