# 办公小浣熊工作流说明

## 触发方式

Star-DART OPC 采用定时轮询机制，而不是外部 webhook 唤醒办公小浣熊：

- 默认轮询间隔：10800 秒，即 3 小时。
- 用户可通过 `.env` 中的 `POLL_INTERVAL` 调整。
- 轮询发现新增 Star 后，生成办公小浣熊任务说明。

采用定时轮询的原因：

1. GitHub Star 列表适合通过 API 增量检查。
2. GitHub 个人 Star 事件没有稳定通用的外部 webhook 入口。
3. 办公小浣熊当前可见能力中包含定时任务、本地 Agent 和数据源，但没有明确展示对外 webhook 唤醒入口。
4. 轮询频率可控，默认 3 小时能平衡实时性和 API 成本。

## 办公小浣熊参与链路

### 1. 本地 Agent

接收 Star-DART 生成的任务说明，读取本地样例、README、项目档案和资产台账。

### 2. Skill 固化流程

将“读取资料 -> 项目研判 -> 生成档案 -> 更新台账 -> 输出周报/PPT”固化为可复用技能。

### 3. research-synthesis

对项目进行结构化分析：

- 项目定位
- 目标用户
- 核心亮点
- 技术架构
- 使用边界
- 复用价值

### 4. docx/pdf

输出正式项目档案，适合进入知识库或作为评审材料附件。

### 5. xlsx

维护结构化项目资产台账，字段包括分类、标签、复用等级、推荐理由、适用场景。

### 6. data-dashboard

基于台账生成可浏览看板：

- 分类分布
- 新增趋势
- 高复用项目
- 待深入项目

### 7. pptx / 专家团

生成技术雷达汇报或参赛演示 PPT，展示从收藏到项目资产的完整闭环。

### 8. 知识库和数据源

将项目档案、周报、台账放入云上知识库、本地文件、飞书或 Obsidian，供后续自然语言检索。

### 9. 定时任务

每周触发“开源情报简报”：

- 汇总本周新增 Star。
- 选出最值得深入研究的项目。
- 生成周报和汇报材料。

## 参赛展示建议

1. 展示办公小浣熊技能页，说明用到了文档、表格、看板、PPT 和研究能力。
2. 展示专家中心，说明用专家团模拟多人协作。
3. 展示数据源页，说明本地文件、飞书和 Obsidian 都可作为知识来源。
4. 展示定时任务页，说明 Star-DART 是持续工作流。
5. 运行 dry-run 命令，展示新增 Star 如何转化为办公小浣熊任务。
