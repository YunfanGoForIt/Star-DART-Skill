# 中文文档模板 - 重点提炼版

## 文档目标

**不是简单翻译 README，而是提炼项目的核心价值：**
- 这个项目解决了什么问题？
- 有什么技术创新或产品亮点？
- 为什么值得 star？（用户/开发者角度）

## AI 精炼提示词

### 阶段一：阅读材料

**必须查看**：
1. `overview.md` - DeepWiki 生成的项目概述
2. `docs/*.md` - 其他相关文档
3. `README.md` - 原始 README

**提取信息**：
- 项目核心功能
- 技术架构
- 解决的问题
- 差异化特点

### 阶段二：生成文档

**写作要求**：
1. **不是翻译**：提炼核心，不是逐句翻译
2. **突出亮点**：为什么值得 star？
3. **技术深度**：架构设计 + 技术创新
4. **实用导向**：使用场景 + 对比分析

**禁止**：
- 简单罗列功能点
- 逐句翻译 README
- 缺少技术分析
- 没有项目定位描述

## 检查清单

生成后检查：
- [ ] 有项目定位描述（一句话）
- [ ] 说明了为什么值得 star
- [ ] 包含架构设计和核心技术点
- [ ] 有使用场景和快速开始
- [ ] 文档结构清晰，层次分明
- [ ] 技术术语保留英文（如变量名、函数名）
